EXAMPLE QUESTIONS
================================================================================

The client includes test questions:
    • "TOP 20 schools with most draft picks"
    • "game_pk, home team name, away team name for all games on 2025-06-30"
    • "How many games did Giants play Dodgers in 2025?"
    • "TOP 20 Home States with most draft picks"

Your questions are processed through:
    1. Cache check (instant if cached)
    2. Example matching (fast if similar)
    3. LLM generation (slower, more flexible)
    4. SQL execution (database query)
    5. Result return & export

================================================================================
USING IN YOUR CODE
================================================================================

Simple usage:
    from mlb_ai_client import MLBQueryClient
    
    client = MLBQueryClient()
    result = client.ask("TOP 10 home run hitters")
    
    print(f"SQL: {result.sql}")
    print(f"Rows: {result.row_count}")
    for row in result.rows:
        print(row)

Batch processing:
    questions = ["Question 1", "Question 2", "Question 3"]
    batch = client.ask_batch(questions)
    print(batch.summary())
    
    # Export results
    from mlb_ai_client import ResultExporter
    exporter = ResultExporter()
    exporter.export_json(batch)
    exporter.export_csv(batch)
    exporter.export_markdown(batch)

================================================================================
OPTIONAL: METADATA FILES
================================================================================

For better results, create metadata/ folder with JSON files:

metadata/examples.json:
    {
        "examples": [
            {
                "question": "TOP 20 schools with draft picks",
                "sql": "SELECT TOP 20 school__name, COUNT(*) FROM dw.draft GROUP BY school__name"
            }
        ]
    }

metadata/rules.json:
    {
        "rules": ["Always use SELECT TOP (100)"],
        "common_mistakes": ["Don't join on scores"]
    }

These speed up queries and improve accuracy.

================================================================================
TROUBLESHOOTING
================================================================================

Server won't start?
    → Check mlb_agent.log
    → Verify SQL Server is running
    → Make sure Ollama is serving

Connection refused?
    → Is server running on :8000?
    → Check: curl http://127.0.0.1:8000

LLM timeout?
    → Verify Ollama: ollama serve
    → Check system resources (CPU/RAM)

No results?
    → Question might be too broad
    → Check query_results/*.md for SQL
    → Review metadata hints
